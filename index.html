¿Qué es Git?

¿Por qué es necesario?

Git es fundamental para cualquier desarrollador, ya que permite controlar las versiones de un proyecto de manera eficiente. Esto evita la pérdida de trabajo, facilita la colaboración en equipo al permitir que múltiples personas trabajen en el mismo código sin sobrescribirlo, y permite revertir a versiones anteriores si se comete un error. Es, en esencia, la columna vertebral de un desarrollo de software organizado y profesional.

Funcionalidades Principales

Ramificación (Branching)

Permite crear ramas de trabajo aisladas para experimentar con nuevas características sin afectar el código principal.

Historial de Versiones

Cada cambio se registra con un "commit", lo que crea un historial completo y navegable del proyecto.

Colaboración Sencilla

Facilita la fusión de los cambios de diferentes desarrolladores de manera controlada y sin conflictos.

Reversión de Cambios

Puedes deshacer cambios o volver a una versión anterior del proyecto en cualquier momento.

FUNCIONAMIENTO

Un flujo de trabajo común para un nuevo proyecto se desglosa en los siguientes pasos:

Paso 1: Clonar el Repositorio 
git clone https://github.com/tu-usuario/nombre-del-repo.git

Copia el repositorio remoto a tu máquina local.

Paso 2: Navegar al Directorio 
cd nombre-del-repo

Entra al directorio de tu proyecto.

Paso 3: Agregar Cambios 
git add .

Agrega los archivos modificados o nuevos al "staging area".

Paso 4: Guardar los Cambios (Commit) 
git commit -m "feat: Agrega un nuevo archivo"

Guarda una nueva versión de tu proyecto con un mensaje descriptivo.

Paso 5: Enviar los Cambios (Push) 
git push origin main

Sube tus cambios al repositorio remoto en GitHub.

Manual de GitHub

GitHub es una plataforma de hosting para repositorios de Git. Es una red social para desarrolladores donde puedes almacenar tus proyectos, colaborar con otros y mostrar tu trabajo al mundo.

Planes de GitHub

GitHub ofrece varios planes, pero para la mayoría de los desarrolladores que trabajan en proyectos personales, el **plan gratuito** es más que suficiente. Este plan incluye:

  • **Repositorios públicos y privados ilimitados:** Puedes crear cuantos proyectos necesites, ya sean visibles para todos o solo para ti y tus colaboradores.
  • **Colaboración en equipo:** Permite que hasta 3 colaboradores trabajen contigo en repositorios privados.
  • **GitHub Pages:** Te permite alojar sitios web estáticos directamente desde tus repositorios de forma gratuita.

Guía Paso a Paso para Iniciar en GitHub

Paso 1: Entrar a GitHub

  • Para empezar, ve a la página de inicio de GitHub. Puedes buscar "GitHub" en Google o escribir la dirección directamente: **https://github.com**.
  • Una vez en la página, busca el botón que dice **"Sign up"** (o "Registrarse") en la esquina superior derecha. Haz clic en él.

Paso 2: Registrarse y Rellenar los Datos

Ahora, debes crear tu perfil. Te pedirán algunos datos importantes para tu cuenta:

  • **Dirección de correo electrónico:** Es muy importante que sea un correo que uses a menudo, ya que lo necesitarás para verificar tu cuenta y recibir notificaciones.
  • **Contraseña:** Elige una contraseña que sea fácil de recordar para ti, pero difícil de adivinar para otros. Debe ser de al menos 8 caracteres.
  • **Nombre de usuario:** Este será el nombre que te identifique en GitHub. Piensa en un nombre que sea profesional, ya que será parte de la dirección de tus proyectos (ej: `github.com/tu-usuario/mi-proyecto`).
  • **Preferencias de correo:** Puedes elegir si quieres que GitHub te envíe correos con noticias y actualizaciones. Es opcional.

Paso 3: Activar la Cuenta

Para confirmar que tu correo es real, GitHub te enviará un **código de seguridad**.

  • **Revisa tu correo:** Ve a la bandeja de entrada del correo que usaste para registrarte. Puede que el mensaje esté en la carpeta de "Spam" o "Correo no deseado", ¡así que revisa también allí!
  • **Ingresa el código:** Copia el código que te enviaron y pégalo en la página de registro de GitHub.

Después de esto, ¡tu cuenta estará activada y lista para usarse!

Paso 4: Crear tu Primer Repositorio

Un **repositorio** es como una carpeta especial donde guardarás tu proyecto. Para crear uno, sigue estos pasos:

  • Inicia sesión en tu cuenta. En la esquina superior derecha, haz clic en el signo **'+'** y selecciona **"New repository"**.
  • **Nombre del repositorio:** Ponle un nombre que describa tu proyecto (ej: `mi-portafolio`). Lo mejor es usar letras minúsculas y guiones (`-`).
  • **Descripción:** Escribe una frase corta que explique de qué trata tu proyecto.
  • **Visibilidad:** Elige si será **Público** (cualquiera puede verlo) o **Privado** (solo tú y los colaboradores que elijas pueden verlo).
  • **README:** **Activa la casilla** "Add a README file". Este archivo es importante porque es la portada de tu proyecto y sirve para dar información sobre él.

Cuando termines, haz clic en el botón verde **"Create repository"** y ¡listo! Tu primer repositorio está creado.

Paso 5: Conectar GitHub con tu Computadora

Para trabajar en tu proyecto desde tu computadora, debes **clonar** el repositorio. Es como hacer una copia local.

  • Ve a la página de tu repositorio en GitHub y haz clic en el botón verde **"<> Code"**.
  • Asegúrate de que la pestaña **HTTPS** esté seleccionada y copia la dirección que aparece.

Ahora, en tu terminal o Git Bash, escribe este comando (reemplazando la dirección con la que copiaste): 

git clone [la_direccion_que_copiaste]

Esto descargará el repositorio a tu computadora, y podrás ver los archivos del proyecto en una carpeta nueva.

Paso 6: Guardar tus Cambios en la Computadora

Cada vez que hagas cambios en tu proyecto (ej: añadas un archivo o cambies algo en el código), debes guardarlos localmente usando dos comandos: 

git add .

Este comando prepara todos los archivos modificados para ser guardados. (La etapa de "staging")

git commit -m "feat: Agrega mi primer archivo HTML"

Con este comando, creas una "fotografía" de tu proyecto en este momento y le pones un mensaje. Es como un punto de guardado.

Paso 7: Subir tus Cambios al Repositorio

El último paso es enviar los cambios que guardaste en tu computadora a GitHub.

  • Abre tu terminal o Git Bash y usa este comando:
    • git push origin main
  • **`git push`** significa "empujar" los cambios.
  • **`origin`** es el nombre de tu repositorio en GitHub.
  • **`main`** es el nombre de la rama principal de tu proyecto.

¡Y listo! Ahora tus cambios estarán visibles en la página de tu repositorio en GitHub para que cualquiera los vea.
